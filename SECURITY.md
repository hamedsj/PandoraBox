# Security Policy

PandoraBox is security tooling. Reports about vulnerabilities in PandoraBox
itself are handled privately; reports about third-party misuse of a deployed
PandoraBox instance are abuse reports and should be routed through the relevant
service provider or organization.

## Supported Versions

Only the latest public release receives security fixes.

## Reporting a Vulnerability

Email hsjbb123@gmail.com with:

- A clear description of the vulnerability.
- Steps to reproduce it from a clean checkout or release build.
- Impact and any known prerequisites.
- A minimal proof of concept, if one is needed to understand the issue.

Please do not open a public GitHub issue for vulnerabilities in PandoraBox.

## Disclosure Timeline

The maintainer aims to acknowledge reports within 7 days and resolve confirmed
issues within 90 days. If a fix needs more time, the maintainer will coordinate
with the reporter before disclosure.

## Abuse Reports

PandoraBox can intercept, replay, and modify traffic. Misuse reports about a
person or organization running PandoraBox against systems they do not own are
not vulnerabilities in this repository. Send those reports to the affected
provider, platform, employer, or legal contact.
